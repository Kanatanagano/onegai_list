# Recursion チーム開発の詳細(Golang webAPI)

## 概要
Recursionでは、学生が実践的なチーム開発を通じてスキルを習得し、企業で即戦力として活躍できるようにします。

本ドキュメントでは、実際のチーム開発での開発フロー、チームのやりとり、期間、成果物、レベル、技術選定、メンターの役割、Git管理方法、オンラインでのやり方、タスク管理、要件定義と開発の自由度、そして取得したスキルセットについて詳しく説明します。

## 開発フロー

1. **要件定義**
   - チームリーダーを中心にコアタイムと製作物の決定
   - 期間内に達成するべきことを羅列
   - 外部webAPIについてキャッチアップ
   - 個人でGolangを使った小さいデモアプリケーションを作成する

2. **技術選定**
   - フロントエンドをHTML/CSS&JavaScript
   - バックエンドをGolang(GIN)
   -  データベースをMySQL

3. **設計**
   - 設計についてチームに共有
   - HTTPリクエストとレスポンスの形式を確認
   - フォルダ形式、コードルールについてチームに共有

4. **開発**
   - タスクに分割し、各メンバーが担当部分を実装
   - コミット内容は第三者にわかりやすいよう
   - プルリクエストを通じてコードレビューを実施


5. **レビューと改善**
   - 開発終了後の振り返り
   - プロジェクトのフィードバック

## 説明
チーム開発ではプログラミング初心者～中級者まで様々なバックグラウンドをもった方がいらっしゃいます。

今回のGolang webAPIではメンバー全員がGolang未経験だったため、簡単なリクエスト・レスポンスをするウェブサイトを作成してその後JSON形式で同じように応答をするウェブサイトを作成しました。

最終的に私たちのチームでは開発生産性に焦点を当てHTTPメソッドに即したコードを書きたかったため、そのコードが比較的書きやすいウェブフレームワークGINを使ってバックエンドのコードをGolangからGolang(GIN)に換えて開発を行っていきました。

このようにチームレベルに合わせて難易度を上げていき最終的な成果物を作成していきました。

## チームのやりとり

- **定期ミーティングの内容**
  - メンバーが開発しているタスクについて進捗確認、Oviceの画面共有機能で挙動確認
  - 開発方針の共有
  - 質疑応答
  - リーダーは開発議事録で今後の方針を簡単にまとめる
  - 次回ミーティングの日程確認

- **コミュニケーションツール**
  - Oviceを利用したリアルタイムコミュニケーション
  - GitHub Issuesを使ったタスク管理
  - discordで質問や進捗を管理

## 説明
ミーティングではメンバーが担当している機能について挙動を確認しました。その後メンバーと追加機能のディスカッションをして担当する部位について決めました。

私たちのチームではコアタイムを導入してその時間ではOvice内で集まって開発するようにしました。開発の難易度が上がると特に質問が多くなっていきタスクを効率的に変更したり、わからないことは実際のコードを画面共有しながら解決していきました。

これらのツールを利用したことにより開発のスピードを上げることができました

## 期間、成果物、レベル、技術選定

- **期間**
  - 1つのプロジェクトについて2週間

- **成果物**
  - 独自のwebAPIを作成したレシピ検索ウェブアプリケーション

- **レベル**
  - 初心者から中級者向けのプロジェクト

- **技術選定**
  - フロントエンドをHTML/CSS&JavaScript
  - バックエンドをGolang
  - データベースをMySQL

## メンターの役割

- **メンターのサポート**
  - コードレビュー
  - 問題解決のサポート
  - プロジェクトの進捗管理とアドバイス

- **教える内容**
  - Gitを使ったソースコードの管理方法
  - チーム開発におけるコーディング規約とベストプラクティス
  - システム設計とスケーラビリティの考え方
  - コミュニケーションスキルとチームワークの重要性

## 説明
メンターは学習者がスムーズに成長できるようにサポートし、技術的な知識やスキルの向上だけでなく、エンジニアとしてのマインドセットやソフトスキルの向上も促進します。

## Git管理方法

- **ブランチ戦略**
  - Gitflowを採用
  - `feature`, `develop`, `release`, `hotfix`ブランチを活用

- **プルリクエスト**
  - プルリクエストを作成してコードレビューを実施
  - レビュワーはメンターおよびチームメンバー
